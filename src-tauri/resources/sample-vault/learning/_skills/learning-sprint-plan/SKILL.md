# Learning sprint plan

Run before starting any new course or topic (data/courses.csv).

1. **Artifact first.** Define what gets BUILT or produced by week 4: a project,
   a written explainer, a working demo. Consumption without an artifact decays
   in weeks.
2. **Scope to fit.** Pick the 20% of the material that serves the artifact.
   Completeness is the enemy of finishing.
3. **Schedule.** Three sessions a week on the calendar (coordinate with the
   calendar domain's protected blocks). Less than that is a hobby, which is
   fine, but name it.
4. **Exit test.** Week 4: artifact shipped? If not, one honest line on why,
   then either one more week or a clean stop. No zombie courses.

Output: the artifact definition, the scoped syllabus, and the session schedule.
